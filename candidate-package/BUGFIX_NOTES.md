# Bug-Fix Notes

> **How to use this file (candidate instructions)**
>
> The starter ships with six documented, intentionally-seeded bugs: `BUG-01` through `BUG-06`.
> Their observable symptoms are listed in [`BUGS.md`](./BUGS.md) (symptom-only — no root cause or fix).
>
> For **each** bug you fix:
> 1. Fill in the matching section below — replace every `TODO` placeholder with your own findings.
>    - **Root cause**: what was actually wrong in the code (file + reason), not just the symptom.
>    - **Applied fix**: the change you made to correct it.
>    - **AI-assisted vs hand-written split**: which parts of the investigation/fix were produced with
>      AI tooling (the agent, Skills, Hooks, MCP, Steering) and which parts you wrote by hand. Be specific.
>      If no AI tooling was used for a bug, write "None — hand-written" explicitly.
> 2. Commit the fix as its **own separate** [Conventional Commit](https://www.conventionalcommits.org/)
>    using the `fix:` type and referencing the bug identifier, for example:
>    ```
>    fix(auth): persist tokens to secure storage so session survives restart (BUG-01)
>    ```
>    Keep one bug per commit so each fix is a coherent, self-contained change.
>
> Leave a bug's section as `TODO` only if you did not fix it.
